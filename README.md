# matcraft

CLI-инструмент для исследования морфологической парадигмы русского мата.

Показывает полное комбинаторное пространство: все возможные комбинации корней,
префиксов, суффиксов и окончаний с указанием степени аттестации (common, rare,
possible). Никакого AI, никаких словарей — чистая комбинаторика + лингвистические
правила.

Вывод — **объясняющий**: каждая словоформа сопровождается разбором на морфемы с
глоссами, уровнем аттестации и объяснением, почему форма именно такого уровня. Весь
вывод на русском.

## Установка

Готовые бинарники под каждую ОС лежат на странице релизов — Rust ставить не нужно:

**<https://github.com/aadegtyarev/matcraft/releases>**

Скачайте архив для своей платформы с последнего релиза (файлы вида
`matcraft-vX.Y.Z-<таргет>.tar.gz` / `.zip`), распакуйте и положите `matcraft` в каталог
из `PATH`.

### Linux (x86_64)

Бинарник статический (musl) — работает на любом дистрибутиве, без зависимостей.

```sh
tar xzf matcraft-vX.Y.Z-x86_64-unknown-linux-musl.tar.gz
mv matcraft ~/.local/bin/    # или /usr/local/bin (нужен sudo)
matcraft random
```

Убедитесь, что каталог (`~/.local/bin`) есть в `PATH`.

### macOS

Скачайте нужный архив: Intel — `x86_64-apple-darwin`, Apple Silicon (M1/M2/…) —
`aarch64-apple-darwin`.

```sh
tar xzf matcraft-vX.Y.Z-aarch64-apple-darwin.tar.gz
mv matcraft /usr/local/bin/
matcraft random
```

Бинарник **не подписан**, поэтому при первом запуске Gatekeeper его заблокирует
(«matcraft cannot be opened because the developer cannot be verified»). Снимите
карантин:

```sh
xattr -d com.apple.quarantine ./matcraft
```

Либо один раз откройте через Finder: правый клик по `matcraft` → **Open** →
подтвердить.

### Windows (x86_64)

Скачайте `matcraft-vX.Y.Z-x86_64-pc-windows-msvc.zip`, распакуйте `matcraft.exe`
в любой каталог и добавьте этот каталог в `PATH`. Затем в PowerShell или cmd:

```powershell
matcraft random
```

Бинарник **не подписан** — при первом запуске SmartScreen покажет предупреждение
(«Windows protected your PC»). Нажмите **«Подробнее»** → **«Выполнить в любом
случае»**.

### Проверка контрольной суммы (необязательно)

Рядом с каждым архивом лежит файл `.sha256`. Проверить целостность скачанного архива:

```sh
sha256sum -c matcraft-vX.Y.Z-x86_64-unknown-linux-musl.tar.gz.sha256
```

(на macOS — `shasum -a 256 -c …`, на Windows — `CertUtil -hashfile … SHA256`).

Если нужна установка из исходников или сборка под другую платформу — см.
[Для разработчиков](#для-разработчиков).

## Использование

### Исследовать парадигму корня

```sh
# Полная парадигма корня еб-
matcraft explore еб-

# Фильтр по суффиксу (значение с ведущим дефисом — через пробел или через =)
matcraft explore еб- --suffix -ну-
matcraft explore еб- --suffix=-ну-
```

Полная парадигма → легенда морфем + обзорная таблица, сгруппированная по суффиксу
(колонки: инфинитив, прошедшее, настоящее 3 л., аттестация, значение). С фильтром
`--suffix` → та же таблица + полный блок-разбор на каждую форму:

```text
осснуть  ·  possible
  значение : — (в источнике не засвидетельствовано)
  разбор   : о-/об- + сса- + -ну- + -ть
  морфемы  : приставка «о-/об-» · корень сса- «мочиться» · суффикс -ну- (однократный) · инфинитив -ть
  уровень  : possible — словообразовательно возможна по аналогии, в источнике не засвидетельствована
  домен    : экскреторная · продуктивность B
```

### Сгенерировать случайную форму

```sh
# Одна форма
matcraft generate --root еб-

# Несколько форм
matcraft generate --root еб- --count 3

# Из любого корня (в текущем режиме)
matcraft generate --count 5
```

Каждая форма выводится полным блоком-разбором (как выше), блоки разделены пустой
строкой. **Изменение вывода (v0.8.0):** `generate` больше не печатает одну форму на
строку — скрипты вида `matcraft generate --count N | wc -l` сломаются.

Для чисто именного корня (глагольных форм нет) `generate --root` печатает
информативное сообщение и завершается с кодом 0 — это корректный ввод, не ошибка.

### Случайный корень с заметкой

```sh
matcraft random
```

Вывод: рамка-сводка корня (русский глосс, домен, продуктивность, суффиксальные классы,
2–3 примера, лингвистическая заметка) + под ней `Пример разбора:` с полным
блоком-разбором для одной образцовой формы.

### Корень дня

```sh
matcraft root-of-the-day
```

Тот же формат рамки, что и `random`, но **детерминированный в пределах суток**: любой
повторный вызов в те же календарные сутки (UTC) даёт тот же корень; со сменой суток
корень меняется.

### Список корней

```sh
# 9 корней в classic-режиме (по умолчанию)
matcraft list-roots

# Все 35 корней
matcraft list-roots --mode full
```

### Режимы

| Флаг | Описание |
|------|----------|
| `--mode classic` | 9 корней: всё ядро (7) + два продуктивных экскреторных (сра-, сса-) — по умолчанию |
| `--mode full` | Все 35 корней, сгруппированы по домену (ядро / экскреторная / периферия) |

`explore <корень>` работает независимо от режима: можно исследовать `сра` в
classic-режиме, хотя он не показывается в списке.

### Аттестация

| Уровень | Значение |
|---------|----------|
| common   | Широко используется, любой носитель знает |
| rare     | Встречается в речи, но не общеупотребительно |
| possible | Грамматически возможно, но в речи не зафиксировано |
| impossible | Блокировано фонологическим или семантическим ограничением |

Уровни аттестации **заземлены на экспертный синтез Плуцера-Сарно**: то, что
помечено common/rare, эксперт извлёк из опубликованных томов; possible — честная
экстраполяция по аналогии, а не выдумка. Методология и полный инвентарь —
`docs/decisions/plutser-sarno-taxonomy.md`.

Объясняющий вывод делает возможные формы **самообъясняющимися**: строка `уровень`
прямо говорит, что `possible`-форма словообразовательно возможна, но в источнике не
засвидетельствована. Это касается и «уродливых» кластеров, которые движок пока строит
(напр. `осснуть`, `вссснёт`): они помечены `possible` с объяснением. Улучшенный вывод
**не исправляет** их построение — морфонологический фикс отслеживается отдельно (#28).

## Для разработчиков

### Сборка из исходников

Если нужна установка из исходников или сборка под платформу, для которой нет
готового бинарника, соберите крейт локально. `matcraft` пока не публикуется в
crates.io — сборка выполняется из репозитория.

#### 1. Установить Rust и cargo

Если Rust ещё не установлен, поставьте его через rustup:

```sh
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

После установки перезапустите шелл или подтяните окружение в текущей сессии:

```sh
source "$HOME/.cargo/env"
```

Проверьте, что cargo доступен:

```sh
cargo --version
```

Требуется Rust ≥ 1.85 (edition 2024) — rustup ставит актуальную stable, этого достаточно.

#### 2. Клонировать репозиторий

```sh
git clone https://github.com/aadegtyarev/matcraft.git
cd matcraft
```

#### 3. Установить бинарник

```sh
cargo install --path .
```

Бинарник ставится в `~/.cargo/bin`, который уже добавлен в `PATH` после установки rustup.

#### 4. Проверить запуск

```sh
matcraft random
```

Требуется Rust 1.85+ (edition 2024).

```sh

```

## Архитектура

```
src/
  main.rs              # CLI (clap, 5 subcommands + --mode)
  engine/
    mod.rs             # Re-exports
    morpheme.rs        # Типы + таблицы морфем (префиксы, суффиксы, окончания)
    grammar.rs         # Конструкция форм (алломорфия, ъ-вставка)
    paradigm.rs        # Построение парадигмы, generate, list-roots
    display/           # Форматирование вывода (общий блок-разбор + по команде)
      mod.rs           # Ре-экспорт + общие хелперы (wrap_text, аттестация, домен)
      form_block.rs    # Общий блок «полный разбор формы»
      explore.rs       # Обзорная таблица + блоки под фильтром
      list.rs          # Список корней
      boxed.rs         # Рамка random / root-of-the-day + пример разбора
    roots/             # Инвентарь корней + таблицы аттестации (один дом на корень)
      mod.rs           # all_roots(), root_data(), lookup_attestation()
      group1.rs        # 7 корней ядра
      group2.rs        # 2 самых продуктивных экскреторных корня
      group3.rs        # 9 глагольных корней (периферия + экскреторные)
      group4.rs        # 5 именных корней с глагольным потенциалом
      group5.rs        # 12 чисто именных корней
```

Подробнее: `docs/architecture.md`

## Источники

Инвентарь, семантическая типология и уровни аттестации заземлены на первоисточник:

- Плуцер-Сарно, А. **«Большой словарь мата».** Том 1 «Хуй», том 2 «Пизда».
  СПб.: Лимбус Пресс, 2001. Введение, строки 864–870.

Эксперт прочитал опубликованные тома и выдал синтез (корпусную проработку);
matcraft грундится на этом синтезе. Курированный справочный документ с полным
инвентарём, доменами, классами продуктивности и методологией аттестации —
`docs/decisions/plutser-sarno-taxonomy.md`.

То, что помечено common/rare, восходит к экспертному синтезу; possible — честная
словообразовательная экстраполяция по аналогии, в источнике не засвидетельствованная.

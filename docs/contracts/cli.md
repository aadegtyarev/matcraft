# Контракт CLI matcraft

Пользовательская поверхность бинарника `matcraft`. Что гарантируется тому,
кто вызывает инструмент; изменение семантики ниже — изменение контракта (semver).
Данные (инвентарь, домены, аттестация) — `docs/decisions/plutser-sarno-taxonomy.md`.

## Принцип вывода: объясняющий, исследовательский

Вывод всех команд — **объясняющий**: каждая словоформа сопровождается
разбором на морфемы с глоссами (русскими), уровнем аттестации и объяснением, почему
форма именно такого уровня. Голого списка форм нет ни в одной команде. Весь текст
вывода — на русском (английский глосс корня — только внутренний dev-ориентир, в вывод
не идёт). Токены уровней (`common`/`rare`/`possible`/`impossible`) оставлены английскими
как пометы источника.

**Общий блок «полный разбор формы»** (переиспользуется в `generate`, `explore` под
фильтром, `random`, `root-of-the-day`):

```text
обосснуть  ·  possible
  значение : — (в источнике не засвидетельствовано)
  разбор   : о-/об- + сса- + -ну- + -ть
  морфемы  : приставка «о-/об-» · корень сса- «мочиться» · суффикс -ну- (однократный) · инфинитив -ть
  уровень  : possible — словообразовательно возможна по аналогии, в источнике не засвидетельствована
  домен    : экскреторная · продуктивность B
```

Форма без пометы источника (`possible`) печатает `значение : —` — значение не
выдумывается (§7 честность). Голый префикс `(без)` в разборе и морфемах опускается.

## Глобальный флаг

| Флаг | Значения | По умолчанию | Семантика |
|---|---|---|---|
| `--mode` | `classic` \| `full` | `classic` | Набор корней для `list-roots`, `generate` (без `--root`), `random`, `root-of-the-day`. Ограничен clap `value_enum`; иное значение → ошибка парсинга (exit 2). |

`--mode` помечен `global = true` в clap: принимается в любой позиции — до или после субкоманды (напр., `matcraft list-roots --mode full`).

`classic` = ядро (7) + два продуктивных экскреторных корня (сра-, сса-) = 9 корней.
`full` = все 35 корней. Состав classic выводится из таксономии
(`Nuclear` ∪ {`Excretory` ≤ B}), а не хардкодится.

## Субкоманды

### `explore <корень> [--suffix <s>]`
Полная комбинаторная парадигма корня (18 префиксов × суффиксы × окончания).

- Работает для **любого** из 35 корней независимо от `--mode`.
- Завершающий дефис в корне отбрасывается (`еб-` = `еб`).
- Заголовок: корень, русский глосс, домен, класс продуктивности, лингвистическая заметка.
- **Полная парадигма (без `--suffix`)** → однократная **легенда морфем** (суффиксы с
  глоссами, окончания-колонки, префиксы) + **обзорная таблица**, сгруппированная по
  суффиксу: одна строка на комбинацию префикс×суффикс, окончания вынесены в колонки
  `Префикс | Инфинитив | Прош. -л | Наст. 3л | Аттест. | Значение` (видны все формы —
  инфинитив, прошедшее, настоящее 3 л. — но строк ~36, а не ~108). Поитоговые и общий
  tally по уровням.
- **С фильтром `--suffix`** (узкий результат) → та же обзорная таблица (короткая) +
  секция `Разбор форм:` с полным блоком-разбором на каждую комбинацию (репрезентант —
  инфинитив).
- Чисто именной корень (нет глагольных форм): версионно-нейтральное сообщение об
  отсутствии глагольной парадигмы + лингвистическая заметка.
- `--suffix` фильтрует по значению или отображению суффикса; невалидное значение
  просто не даёт совпадений (не ошибка) — для глагольного корня выводится
  «Нет форм по заданному фильтру.» (exit 0), а не ложная пометка «именной корень».
  Принимает значения с ведущим дефисом в обеих формах: `--suffix -ну-` (через пробел)
  и `--suffix=-ну-` (через `=`).
- Неизвестный корень → stderr с сообщением и списком доступных корней, **exit 1**.

### `generate [--root <корень>] [--count <N>]`
Случайные словоформы из пула (аттестация ≠ `Impossible`).

- **Каждая форма — полный блок-разбор** (общий блок выше), блоки разделены пустой
  строкой. **Изменение контракта вывода:** `generate` больше не печатает
  одну-форму-на-строку — скрипты `matcraft generate --count N | wc -l` сломаются.
- Без `--root`: пул из корней текущего `--mode`.
- `--count` по умолчанию 1; **> 100 → exit 1**; значение внутренне зажато в 1..=100.
- Неизвестный `--root` → **exit 1**.
- Валидный чисто именной `--root` (нет глагольных форм) → информативное сообщение на
  stdout («именной корень «<корень>-»: глагольных форм нет») и **exit 0**: это
  корректный ввод, а не ошибка. Случай без `--root` (пул) сообщением не затронут.

### `list-roots`
Список корней текущего `--mode`. Строка корня: `<корень>-  «<русский глосс>»  <домен> ·
<класс> · <тип>`, где тип = `глагольный (<суффиксальные классы>)` либо `только именной`.
Глагольный корень без единой засвидетельствованной формы (весь paradigm — `possible`)
несёт честную пометку `(нет засвидетельствованных форм)` (§7).

- `full`: сгруппирован по домену (Ядро / Экскреторная / Периферия).
- `classic`: плоский список из 9 корней в том же формате строки.

### `random`
Один случайный корень текущего `--mode`: обогащённая **рамка-сводка** (русский глосс,
домен · продуктивность, суффиксальные классы с глоссами, до 3 примеров-инфинитивов
уровня common, лингвистическая заметка) + под рамкой `Пример разбора:` с полным
блоком-разбором для одной образцовой формы (common-инфинитив). Нет common-инфинитива
(именной корень, или глагольный без засвидетельствованных форм) → честная пометка
вместо выдуманного примера.

### `root-of-the-day`
Один корень текущего `--mode` в том же формате, что `random` (рамка-сводка + `Пример
разбора:`).

- **Детерминирован в пределах календарных суток UTC**: любой повторный вызов в те же
  сутки даёт тот же корень; со сменой суток корень меняется.
- Сутки берутся по **UTC** (секунды системных часов с эпохи / 86400), а не по
  локальному часовому поясу — вблизи локальной полуночи смена корня может быть смещена
  относительно локального дня.
- Конкретное соответствие «день → корень» **не** является гарантией стабильности между
  версиями: сид детерминирован в пределах major-версии `rand`, будущий major может
  перетасовать соответствие. Контракт обещает «один корень в пределах суток», а не
  «день X → корень Y навсегда».
- Аргументов нет; чисто именной корень — без примеров, с пометкой (как `random`).

## Уровни аттестации

`common` · `rare` · `possible` · `impossible` — заземлены на синтез Плуцера-Сарно;
`possible` — честная экстраполяция по аналогии (методология в справочном доке).

## Коды возврата

| Код | Значение |
|---|---|
| 0 | Успех |
| 1 | Ошибка приложения (неизвестный корень, count > 100) |
| 2 | Ошибка разбора аргументов (clap) |

## Стабильность

Инструмент офлайн, без сети, файлов и пользовательских данных. Вывод —
детерминирован, кроме `generate`/`random` (случайный выбор каждый вызов).
`root-of-the-day` — промежуточный случай: детерминирован **в пределах суток**, меняется
со сменой суток UTC. Инвентарь из 35 корней и состав `classic` (9) — стабильный
контракт в пределах minor-версий.

# Архитектура matcraft

## Назначение и область применения

matcraft — CLI-генератор русских матерных выражений по морфологическим правилам.
Предназначен для русскоязычных разработчиков и энтузиастов, интересующихся
программной лингвистикой.

Работает офлайн, не требует внешних API или LLM.

## Стек

- Язык: Rust (edition 2024)
- CLI-фреймворк: clap 4.6 (derive)
- Случайный выбор: rand 0.9
- Сборка: Cargo
- CI: GitHub Actions
- Распространение: crates.io

## Слои системы

```
┌─────────────┐
│    CLI      │ clap — парсинг аргументов, вывод результата
├─────────────┤
│   Engine    │ Модуль морфологической генерации
│             │ Корни, приставки, суффиксы, правила комбинаторики
└─────────────┘
```

### Слой CLI (src/main.rs и будущие модули)

Отвечает за:
- Парсинг аргументов командной строки
- Валидацию темы
- Вывод результата пользователю
- Коды возврата (0 — успех, 1 — ошибка)

### Слой Engine (планируется в src/engine/)

Будет отвечать за:
- Хранение морфем (корни, приставки, суффиксы, окончания, постфиксы)
- Правила комбинирования (какие приставки с какими корнями сочетаются)
- Случайный выбор морфем с учётом весов

## Модульная структура (план)

```
src/
├── main.rs           # Точка входа, CLI
├── engine/
│   ├── mod.rs        # Публичный API engine
│   ├── roots.rs      # Корневые морфемы (хуй, пизд-, еб-, бляд-)
│   ├── affixes.rs    # Приставки, суффиксы, окончания
│   ├── rules.rs      # Правила комбинирования морфем
│   └── generator.rs  # Генератор — выбор и комбинация
└── quality/          # Инструменты контроля качества
    └── tools.json
```

На данный момент (скелет) весь код в `src/main.rs`. Разделение на модули
произойдёт при реализации первого полноценного engine.

## Модель данных (эскиз)

- **Корень (Root):** строка, тема, список совместимых приставок/суффиксов
- **Приставка (Prefix):** строка, список совместимых корней `[?]`
- **Суффикс (Suffix):** строка, тип (глагольный/именной)
- **Постфикс (Postfix):** строка, применяется после суффикса
- **Правило (Rule):** корень + приставка? + суффикс + окончание + постфикс?

`[?]` — полная матрица совместимости приставок с корнями пока не определена.
Требуется лингвистическое исследование.

## Неизвестные вопросы

- Полная матрица совместимости аффиксов (какие префиксы работают с какими
  корнями?) `[?]`
- Таксономия тем (иерархия или плоский список?) `[?]`
- Нужны ли веса для морфем (одни комбинации естественнее других?) `[?]`

## Окружение

- Платформы: Linux, macOS, Windows
- Режим: только офлайн
- Бюджет: отсутствует (open-source, хобби-проект)
- Зависимости от внешних сервисов: нет

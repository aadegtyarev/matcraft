# Архитектура matcraft

## Назначение и область применения

matcraft — CLI-генератор матерных выражений по заданной теме.
Берёт любую тему (слово или фразу) и генерирует обсценные фразы,
комбинируя тему с матерными корнями по морфологическим правилам
русского языка.

Тема используется **как есть** — она становится существительным в каждой
фразе. Разнообразие обеспечивается морфологическим движком:
4 корня, ~3 шаблона фраз, согласование по роду.

Работает офлайн, не требует внешних API или LLM.

## Стек

- Язык: Rust (edition 2024)
- CLI-фреймворк: clap 4.6 (derive)
- Случайный выбор: rand 0.9
- Сборка: Cargo
- CI: GitHub Actions
- Распространение: crates.io

## Слои системы

```
┌─────────────────────────────────────────────────────┐
│                     CLI (main.rs)                    │
│  clap: --topic <СТРОКА> [--count <N>]              │
│  Валидация: тема не пуста, count ≤ 100             │
└─────────────────────┬───────────────────────────────┘
                      │ engine::generate(topic, count)
                      ▼
┌─────────────────────────────────────────────────────┐
│              Engine (src/engine/)                    │
│                                                      │
│  ┌────────────────────────────────────────────┐     │
│  │            generator.rs                    │     │
│  │  • guess_gender(word) → Gender            │     │
│  │  • generate(topic, count) → Vec<String>   │     │
│  │  • template application                    │     │
│  └─────────┬──────────────────┬──────────────┘     │
│            │                  │                     │
│    ┌───────▼───────┐  ┌──────▼────────┐            │
│    │  morphemes.rs  │  │   (data)      │            │
│    │  ROOTS[4]      │  │  INTERJECTIONS│            │
│    │  Gender enum   │  │  EVAL_NOUNS   │            │
│    └───────────────┘  └───────────────┘            │
└─────────────────────────────────────────────────────┘
```

### Слой CLI (src/main.rs)

Отвечает за:
- Парсинг аргументов (--topic, --count)
- Валидацию: тема не пуста
- Вывод результата пользователю
- Коды возврата: 0 — успех, 1 — ошибка, 2 — ошибка аргументов

### Слой Engine (src/engine/)

Состоит из трёх модулей:

#### mod.rs
Публичный API: `engine::generate(topic, count) -> Vec<String>`.
Реэкспорт из generator.rs. Точка входа для CLI.

#### morphemes.rs
Статические данные:

- `Gender` — перечисление: Masculine, Feminine, Neuter, Plural
- `Root` — структура корня: имя, основа прилагательного, окончания по родам
- `ROOTS` — 4 корня: пизд-, хуй-, еб-, бляд-
- `INTERJECTIONS` — общие междометия (7 шт.)
- `EVAL_NOUNS` — общие оценочные существительные (6 шт.)

#### generator.rs
Логика генерации:

1. **guess_gender(word) → Gender** — эвристика по последнему символу слова.
   Для многословных тем анализируется последнее слово.
2. **generate(topic, count) → Vec<String>** — основной генератор: строит
   полный пул всех возможных комбинаций (корень→прилагательное по роду,
   междометие→тема, тема→оценка), перемешивает (shuffle) и выбирает без
   повторений — повторяет цикл, только если count превышает размер пула.
   Такой подход гарантирует максимальное разнообразие. Count ограничен 100.
3. **format_adjective(root, gender) → String** — сборка прилагательного
   из основы корня и окончания по роду.

## Модель данных

### Gender
```
Masculine — нулевое окончание, согласный, й
Feminine  — а, я
Neuter    — о, е
Plural    — ы, и
```

### Root
| Поле | Тип | Назначение |
|------|-----|-----------|
| name | &str | Название корня (для отладки) |
| adjective_stem | &str | Основа прилагательного (без окончания) |
| adjective_endings | [&str; 4] | Окончания: [муж, жен, ср, мн] |

### Шаблоны фраз

Три шаблона, каждый построен из общего пула: все комбинации собираются,
перемешиваются и выбираются без повторений:

1. **AdjectiveNoun** — `"[ПРИЛ] [ТЕМА]"` — прилагательное от случайного корня,
   согласованное по роду с темой. Пример: "пиздатые птички", "хуёвый космос"
2. **Interjection** — `"[МЕЖД], [ТЕМА]!"` — случайное междометие из общего
   списка. Пример: "охуеть, космос!", "пиздец, птички!"
3. **Evaluation** — `"[ТЕМА] — [ОЦЕНКА]"` — тема приравнивается к случайному
   оценочному существительному. Пример: "космос — пиздец", "птички — хуйня"

## Эвристика определения рода

Род существительного определяется по последнему символу:

| Окончание | Род | Примеры |
|-----------|-----|---------|
| а, я | Женский | философия, птичка, рыба |
| о, е | Средний | море, солнце, окно |
| ы, и | Множественное | птички, автомобили |
| согласный, й | Мужской | космос, дятел, чай |
| ь | Мужской* | автомобиль, день |

*Для слов на -ь эвристика даёт мужской род, хотя ~50% таких слов
в русском языке — женского рода (тетрадь, грязь, боль). Это сознательное
ограничение: в контексте мата слова на -ь с женским родом встречаются редко.

**Ограничения:**
- Не учитывает род для заимствований (like, gamepad → мужской по умолчанию)
- Многословные темы: анализируется только последнее слово
- Слова на -ь: ~50% точность

# Архитектура matcraft

## Назначение и область применения

matcraft — CLI-инструмент для исследования морфологической парадигмы русского мата.
Показывает полное комбинаторное пространство префиксов, суффиксов и окончаний
для каждого корня, с указанием аттестации (common, rare, possible, impossible).

Работает офлайн, не требует внешних API или LLM.

## Стек

- Язык: Rust (edition 2024)
- CLI-фреймворк: clap 4.6 (derive)
- Случайный выбор: rand 0.9
- Сборка: Cargo
- CI: GitHub Actions
- Распространение: crates.io

### Слой CLI (src/main.rs)

Отвечает за:
- Парсинг subcommands: `explore`, `generate`, `list-roots`, `random`, `root-of-the-day`
- Глобальный флаг `--mode {classic|full}` (default: classic)
- Валидацию: корень существует (для explore/generate), count ≤ 100
- Вывод результата пользователю
- Коды возврата: 0 — успех, 1 — ошибка

### Слой Engine (src/engine/)

Шесть модулей:

#### mod.rs
Re-exports: `explore()`, `generate()`, `random_root()`, `root_of_the_day()`, `sample_forms()`, типы `Attestation`, `VerbForm`, etc.

#### morpheme.rs
Типы и таблицы морфем (инвентарь корней — в `roots/`):

- `Mode` — режим отображения: Classic (9 корней) / Full (все 35)
- `Domain` — семантический домен (Nuclear / Excretory / Peripheral, источник §1)
- `ProductivityClass` — класс продуктивности A–E (источник §2)
- `Attestation` — 4 уровня: Common, Rare, Possible, Impossible
- `VerbForm` — структура конкретной глагольной формы
- `ParadigmResult` — результат explore()
- `RootData` — данные корня: имя, val, глосс, суффиксы, домен, продуктивность,
  present_stem, заметка
- `PREFIXES` — таблица из 18 префиксов
- `SUFFIXES` — 4 суффикса (-а-, -ну-, -е-, -и-)
- `ENDINGS` — окончания (инфинитив, прош.вр., наст.вр. 3л.)
- Алломорф-селекция: `select_prefix_allomorph()` для из-/ис-, раз-/рас-, вз-/вс-, о-/об-

#### grammar.rs
Правила комбинации морфем в словоформы:

1. **ъ-вставка**: после префиксов от-, раз-, вз-, в-, с-, под-, над-, пред-, об-
   перед корнем на е, ё, ю, я вставляется твёрдый знак (ъ).
2. **Префиксальная алломорфия**: из-/ис-, раз-/рас- (глухость), вз-/вс- (глухость),
   о-/об- (упрощение перед согласным).
3. **Корневая алломорфия**: present_stem для блев-/блю- (чередование ев/у).
4. **Suffix+ending**: таблица комбинаций суффикса и окончания.

#### paradigm.rs
Основная логика:

- `explore(root, suffix_filter)` — строит полную комбинаторную парадигму.
  Для каждого (префикс × суффикс × окончание) создаёт VerbForm.
- `generate(mode, root_opt, count)` — выбирает случайные формы из пула режима.
- `list_roots(mode)` — возвращает список корней, отфильтрованный по режиму.
- `random_root(mode)` — выбирает случайный корень из доступных в режиме.
- `root_of_the_day(mode, day_index)` — детерминированный корень: сид `day_index`
  через `StdRng`, тот же выбор из пула, что `random_root` (общий помощник `select_root`).
- `sample_forms(rd)` — до 3 примеров-инфинитивов уровня common для рамки (общий для
  `random` и `root-of-the-day`).
- Смотрит attestation через `roots::lookup_attestation()`.

#### roots/ (подмодуль)

Инвентарь корней и таблицы аттестации — один дом на корень (определение и
аттестация рядом):

- `mod.rs`: `all_roots()`, `root_data()`, `lookup_attestation()` — агрегация 35
  корней и диспетчеризация аттестации по корню
- `group1.rs`–`group5.rs`: `GROUP_N` (литералы `RootData`) + таблицы аттестации
  (prefix_idx, suffix_idx, attestation, note)

#### display.rs
Форматирование вывода:

- `format_explore(&ParadigmResult) -> String` — таблица с группировкой по суффиксу
- `format_list_roots(mode) -> String` — список корней с группировкой по группам (в full)
- `format_random(&RootData, &[String]) -> String` — корень с рамкой, примерами и заметкой

## Режимы работы

### Mode

Перечисление `Mode` определяет, какие корни видны в списке и участвуют в генерации:

- `Mode::Classic` — 9 корней. Поведение по умолчанию, обратно совместимо с v0.4.0.
- `Mode::Full` — все 35 корней.

Состав classic выводится из таксономии, без хардкода имён: `Nuclear` ∪
{`Excretory` с продуктивностью ≤ B} = ядро(7) + {сра, сса} = 9. Методология —
`docs/decisions/plutser-sarno-taxonomy.md`.

### Domain и ProductivityClass

Две ортогональные оси источника (§1 домен, §2 продуктивность). POS-признак
(«есть глагольная парадигма») выводится из непустоты `suffix_indices`, а не
хранится как отдельная категория.

| Домен (`Domain`) | Корней | Примеры | Продуктивность |
|--------|--------|---------|----------------|
| Nuclear (ядро) | 7 | еб- (A), пизд- (B), хуй- (C) | A–E |
| Excretory (экскреторная) | 7 | сра- (B), перд- (C), говн- (D) | B–D |
| Peripheral (периферия) | 21 | дроч- (D), гондон- (E) | D–E |

Полный инвентарь с доменом, продуктивностью и суффиксами каждого корня —
`docs/decisions/plutser-sarno-taxonomy.md` (one-home).

### Принцип фильтрации

- `explore <корень>` — не фильтруется, работает для любого корня.
- `list-roots` — возвращает корни, где `mode.includes(rd)` истинно.
- `generate` (без --root) — выбирает из корней текущего режима.
- `random` — выбирает из корней текущего режима.
- `root-of-the-day` — выбирает из корней текущего режима (детерминированно по суткам).

## Таксономия корней по Плуцеру-Сарно

Инвентарь и типология заземлены на экспертный синтез Плуцера-Сарно. Единственный
дом инвентаря, доменов, продуктивности и методологии аттестации —
`docs/decisions/plutser-sarno-taxonomy.md`. Здесь — только связь с модулями кода:
`RootData` каждого корня живёт в своём `roots/groupN.rs` рядом с таблицей
аттестации; `all_roots()` агрегирует их в инвентарь из 35 корней.

## Модель данных

### Attestation
```
Common     — широко используется, любой носитель знает
Rare       — встречается в речи, но не общеупотребительно
Possible   — грамматически возможно, но в речи не зафиксировано
Impossible — блокировано фонологическим или семантическим ограничением
```

Уровни аттестации заземлены на экспертный синтез Плуцера-Сарно; методология и
маппинг трёх помёт источника на 4 уровня — `docs/decisions/plutser-sarno-taxonomy.md`.
Аналогическая экстраполяция помечается `Possible`, не `Common`/`Rare`.

### VerbForm
| Поле | Тип | Назначение |
|------|-----|-----------|
| prefix_display | &str | Отображаемое имя префикса |
| suffix_val | &str | Суффикс |
| ending_val | &str | Окончание |
| form | String | Собранное слово |
| attestation | Attestation | Уровень аттестации |
| note | Option<&str> | Заметка о значении |

### Алломорфия

**из-/ис-:** перед глухими согласными и перед корнем еб- используется алломорф
«ис-». Пример: ис- + еб- → исе- (не изъе-). Перед звонкими согласными и гласными
(кроме еб-) используется «из-».

**раз-/рас-:** перед глухими согласными используется «рас-». Пример: рас- + сра-
→ рас-сра- (не раз-сра-). Перед звонкими и гласными — «раз-».

**вз-/вс-:** перед глухими согласными используется «вс-». Пример: вс- + пизд-
→ вс-пизд- (не вз-пизд-). Перед звонкими и гласными — «вз-».

**о-/об-:** перед согласными используется «о-». Пример: о- + пизд- → опизд-.
Перед гласными — «об-». Пример: об- + еб- → объеб-. Это правило **предлога** о/об,
сознательно применённое как **упрощение** к глагольному префиксу: сам глагольный префикс
о-/об- нерегулярен (нерегулярная алломорфия, чистым правилом невыводима), и у него есть
третий алломорф **обо-** (обосраться, обдристать, ободрать), которого движок **не имеет** —
известный gap. Обоснование и источники: `docs/decisions/o-ob-allomorphy.md`.

**от-/ото-:** перед некоторыми формами может использоваться «ото-» (алломорф
ото- в движке не реализован; ъ-вставка применяется к «от-»).

**Корневая алломорфия (блев-/блю-):** корень блев- в презентных формах использует
основу блю- (чередование ев/у). Пример: блевать (инф.), блюёт (наст. 3sg).
В текущем движке реализовано через поле `present_stem: Option<&str>` в RootData.

## Схема комбинаторики

Для корня еб-: 18 префиксов × 2 суффикса (-а-, -ну-) = 36 комбинаций.
Для корня пизд-: 18 префиксов × 3 суффикса (-е-, -ну-, -и-) = 54 комбинации.
Для корня хуй-: 18 префиксов × 1 суффикс (-ну-) = 18 комбинаций.
Для корня дроч-: 18 префиксов × 1 суффикс (-и-) = 18 комбинаций.
Каждая комбинация имеет 3 окончания (инфинитив, прош.вр., наст.вр. 3л.;
для чисто именных корней — 0).

Аттестация по умолчанию: `Possible` — любая незаписанная комбинация считается
грамматически возможной. Это отделяет «реальное» от «возможного» и делает
продуктивность префиксов видимой.

## Неизвестные вопросы

- Инфлекционная парадигма (все лица, все времена) `[?]`
- Именная деривация (пиздатый, хуёвый) `[?]`
- Конфигурационные файлы для определения новых корней `[?]`

Известные gap'ы движка (источник свидетельствует, движок не воспроизводит:
хуярить/хуячить через -j-, блядовать через -ова-, -ва- презентные формы,
домандякаться) перечислены в `docs/decisions/plutser-sarno-taxonomy.md`.

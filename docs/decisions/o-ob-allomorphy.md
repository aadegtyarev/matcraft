# Алломорфия о-/об-/обо- (issue #24)

**Решение.** Правило `select_prefix_allomorph` в движке (о- перед согласным, об- перед
гласным) — это правило **предлога** о/об, сознательно применённое как упрощение к
**глагольному префиксу** о-/об-. Глагольный префикс о-/об- в русском — **нерегулярная
алломорфия**, чистым фонологическим правилом невыводимая, поэтому «починка правилом»
невозможна. Общее правило движка **оставлено без изменений** (по-корневой таблицы
о-/об-/обо- нет); честность аттестации приведена к инварианту §7
(`docs/decisions/plutser-sarno-taxonomy.md`). Отсутствующий алломорф **обо-** зафиксирован
как известный gap движка (для корней-триггеров беглой -о- он теперь строится — issue #28,
`docs/decisions/cluster-fill-vowel.md`). Issue #24 закрыт этим выводом.

**Точечное дополнение (v0.9.0, Оператор подтвердил).** Для корня **говн-** добавлен
минимальный лексический override: флаг `o_takes_ob: bool` (true **только** у говн-)
заставляет «о» arm `select_prefix_allomorph` вернуть «об» и перед согласным, так что движок
строит реальную аттестованную форму **обговнить** (Common), а не упрощённую оговнить. Это
**не** механизм беглой -о- (говн- гласную не вставляет: г — одиночный согласный). Прочие
корни (охарить, одрочить, оебать…) **неизменны**. Кейс говн- по #24 решён; общая
нерегулярность о-/об- для остальных корней остаётся задокументированным упрощением.

## Контекст

Движок строит глаголы: `select_prefix_allomorph` в `src/engine/morpheme.rs` даёт
«об-» только перед гласным корнем и «о-» перед любым согласным. Задокументировано в
`docs/architecture.md` (блок «о-/об-»). Issue #24 спрашивал: верно ли это, или движок должен
строить *обговнить* вместо *оговнить*?

Версия v0.6.1 (`CHANGELOG.md`) сознательно поменяла заметку у *говн-* с «обговнить» на
«оговнить», чтобы она совпадала с формой, которую строит движок, — но оставила метку
`Common`. #24 — отложенный пересмотр именно того решения «по источнику».

## Доказательства (триангуляция, дата проверки 2026-07-06)

1. **Baydimirova A. (2010). «Russian aspectual prefixes O, OB and OBO: a case study of
   allomorphy». MA thesis, University of Tromsø (UiT).**
   <https://munin.uit.no/handle/10037/2767> — *академический источник, высшая ступень.*
   Вывод: распределение O/OB/OBO **не строго дополнительное** («not precisely
   complementary») — это нерегулярная алломорфия; фонологические ограничения есть, но
   допускают широкую вариативность, а выбор лексически и семантически обусловлен.
   **Уверенность: ВЫСОКАЯ.**
2. **Минимальная пара: о-судить ‘осуждать’ ≠ об-судить ‘обсуждать’** — один фонологический
   контекст (корень суд-), разный алломорф, разное значение. Прямое опровержение
   чисто-фонологического правила для глагольного префикса. **ВЫСОКАЯ.**
3. **обо- — третий алломорф**, обязателен перед стечением согласных: ободрать, обозвать,
   обогнать (несколько независимых источников; Кронгауз описывает O- и ОБ- как две разные
   морфемы с общим набором алломорфов о-/об-/обо-, но разной иерархией распределения).
   Движок этого алломорфа **не имеет вовсе**. **ВЫСОКАЯ.**
4. **Предлог vs префикс.** Правило движка (об- перед гласным) совпадает с правилом
   **предлога** о/об (о мне / об арбузе — об- перед а/и/о/у/э), у которого распределение
   действительно чисто фонологическое. Но matcraft строит **глаголы**, где релевантен
   глагольный **префикс** — а он нерегулярен. То есть движок применяет правило предлога к
   префиксу. **ВЫСОКАЯ.**

### Что строит движок vs реальная форма (idx-11, приставка о-)

| Корень | Форма (движок) | Реальная форма (источники) |
|---|---|---|
| хар- | охарить | охарить/обхарить — маргинально, непроверяемо |
| блев- | облевать | облевать ✓ (корень на «б» — совпало случайно) |
| дрист- | одристать | обдристать(ся) (об-/обо-) |
| говн- | обговнить | обговнять/обгавнякать (об-) ✓ — реализовано через `o_takes_ob` (v0.9.0) |
| пизд- | опиздеть | опиздеть (о-) ✓ — чистое правило тут право |
| сра- | обосрать | обосраться (обо- — реализовано через беглую -о-, issue #28) |
| сса- | обоссать | обоссать (обо- — реализовано через беглую -о-, issue #28) |

### Ограничение доказательной базы (явно)

Сырые тома Плуцера-Сарно (том 1 «Хуй», том 2 «Пизда») **в репозитории отсутствуют** —
`plutser-sarno-taxonomy.md` строится на экспертной выжимке, не на парсинге текста. Поэтому
**пословную аттестацию** конкретных о-/об- форм для *говн-*/*хар-* (обгавнякать? обхарить?)
**проверить по первоисточнику нельзя**. Эти конкретные ранжирования помечены как
**unverified** (низкая уверенность), в отличие от лингвистического вывода о нерегулярности
(высокая, академически подтверждённая).

## Что решили

- **Shape A** — общее правило `select_prefix_allomorph` не меняем: чистое фонологическое
  правило в принципе не может быть верным для глагольного префикса, а грунта для пер-корневой
  таблицы алломорфов нет (сырых томов нет). Правило переклассифицировано из
  «неотрефлексированного» в **сознательное задокументированное упрощение** (правило
  предлога). #24 закрыт этим выводом.
- **Точечный лексический override для говн- (v0.9.0)** — флаг `o_takes_ob` (говн-only)
  чинит именно тот корень, ради которого #24 и заводился (обговнить). Это не пер-корневая
  таблица, а минимальное лексическое исключение; форма говн- (11,3) стала реальной
  аттестованной **обговнить**, метка `Common`, заметка совпадает с построенной формой
  (`src/engine/roots/group4.rs`). Прочие корни не тронуты.
- **Правка честности по §7 для дрист-** (только метка аттестации, построенная строка-форма
  не меняется): *дрист- (11,0)* `Rare` → `Possible`, заметка приведена к построенной форме
  `одристать` с пометой о gap-е (`src/engine/roots/group3.rs`). Прежняя заметка «обдристать»
  нарушала политику v0.6.1 «заметка совпадает с построенной формой».
- **обо-** зафиксирован как **известный gap движка** — рядом с ото-, хуярить/хуячить
  (`docs/decisions/plutser-sarno-taxonomy.md`, раздел «Известные gap'ы»); для корней-триггеров
  беглой -о- реализован (issue #28).

## Последствия / что осталось открытым

**Частично закрыто (issue #28).** Для корней-триггеров беглой -о- (сра-, сса-, жр-) алломорф
**обо-** теперь строится (обосрать, обоссать, обожрать) — через отдельный механизм беглой
гласной, а не пер-корневую таблицу о-/об-/обо-. Это сознательно частично разворачивает
отложение #24 для трёх корней; общий обо-gap по прочим корням (дрист-, хар-…)
**остаётся** документированным упрощением (говн- решён отдельно, через об-, см. выше).
Механизм, охват и уверенность: `docs/decisions/cluster-fill-vowel.md`.

Правильные формы (обговнять, обдристать, ободрать) для **прочих** корней потребовали бы
**смены модели**: пер-корневой лексической таблицы алломорфов (какой из о-/об-/обо- берёт
каждый корень) **плюс** реализации отсутствующего алломорфа **обо-** (со взаимодействием с
ъ-/о-вставкой и ретестом всех корней idx-11). Это не «правка правила», а новая работа, и она
**требует грунта по источнику** — то есть доступа к сырым томам для пословной аттестации.
Отложено как потенциальная будущая работа, не реализуется спекулятивно.

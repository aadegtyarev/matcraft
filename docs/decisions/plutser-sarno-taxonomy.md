# Таксономия и аттестация по Плуцеру-Сарно (one-home)

Единственный дом для инвентаря корней, семантической типологии и методологии
аттестации matcraft. Все прочие доки (`architecture.md`, `README.md`, `CHANGELOG.md`)
и комментарии кода **ссылаются сюда**, а не дублируют (инвариант 6). Исполняемая
истина — код в `src/engine/roots/` и `src/engine/morpheme.rs`; этот док — её
человекочитаемое обоснование и методология.

## Библиография

- Плуцер-Сарно, А. «Большой словарь мата». Том 1 «Хуй», том 2 «Пизда».
  СПб.: Лимбус Пресс, 2001. Введение, строки 864–870 (нумерация по .txt-версии).

Синтез — **экспертная выжимка** из опубликованных томов (эксперт прочитал ~12500 +
~11200 строк и выдал приведённую типологию), а не парсинг сырого текста. Сырые тома
в репозитории не нужны. Дата экспертного ответа: 2026-07-05.

## Семантические домены (источник §1)

Источник даёт три домена. Счёт совпадает с инвентарём кода (`Domain` в `morpheme.rs`):

| Домен (`Domain`) | Корней | Описание |
|---|---|---|
| `Nuclear` (ядро) | 7 | Собственно матерное ядро; экспрессивное ядро — хуй+пизда+ебать+блядь |
| `Excretory` (экскреторная) | 7 | Обсценная, автономна от сексуальной, иная система табу/помет |
| `Peripheral` (периферия) | 21 | Сексуальная + именная лексика |

Итого 7 + 7 + 21 = 35.

## Классы продуктивности (источник §2)

`ProductivityClass` A–E, где A — максимальная продуктивность, E — минимальная/нулевая.

- **A** — полная парадигма: *еб-*.
- **B** — широкая (~12 префиксов): *пизд-*, *ср-*, *сс-*.
- **C** — умеренная (~5–10 префиксов): *перд-*, *бзд-*, *дрист-*, *хуj-*.
- **D** — узкая (~2–5 префиксов): *муд-*, *бляд-*, *манд-*, *дроч-*, *трах-*, *хар-*,
  *жр-*, *блев-*, *хер-*, *говн-*, *жоп-*.
- **E** — минимальная/нулевая (0–1 форма, possible): *елд-* и вся именная периферия
  (*залуп-*, *пидор-*, *курв-*, *сиповк-*, *секел-*, *поц-*, *молофь-*, *минж-*,
  *целк-*, *королёвк-*, *кун-*, *сперм-*, *менстр-*, *минет-*, *гондон-*).

*(Прим.: *хер-* источником в §2 явно не отнесён к классу; помечен D по аналогии с
периферийными глагольными корнями — экстраполяция, не корпусное свидетельство.)*

## Методология аттестации

Источник §6 даёт **три пометы**; код имеет **4-уровневый** `Attestation`. Маппинг:

| Источник (§6) | `Attestation` |
|---|---|
| common — широко, многократно в словаре | `Common` |
| rare — единично / окказионально / диалектно | `Rare` |
| possible — возможна по аналогии, в корпусе не отмечена | `Possible` |
| (нет пометы; реальный фоно/семантический блок) | `Impossible` |

**Honesty-правило (источник §7).** Словарь — фразеологический, не словообразовательный:
фиксирует употребление, не кодифицирует продуктивность. Поэтому форма, которую движок
строит, но источник не засвидетельствовал, помечается `Possible` (честная
экстраполяция), а НЕ `Common`/`Rare`. Не помечать possible-форму как attested.

**Per-root политика.** Для *еб-* из-за «местоглагольности» (§4) `Common` допустим даже
на незафиксированное сочетание — почти любой префикс продуктивен. Для остальных корней
разметка консервативна: `Common`/`Rare` только там, где источник свидетельствует
(✓-формы §3 или приставочные дериваты §5).

## Известные gap'ы движка (источник свидетельствует, движок не воспроизводит)

Эти формы источник фиксирует, но текущий комбинаторный движок их не строит; они НЕ
выводятся как ложные формы:

- **хуярить / хуячить** — нужна `-j-` морфонология; *хуй-* оставлен на классе -ну-.
- **блядовать** — суффикс `-ова-`, в движке не реализован.
- **-ва- презентные формы** — вторичный имперфектив; фантомный класс `-ва-` (idx 4)
  удалён из движка (был latent panic-путь). *блев-* моделируется через `-а-` +
  `present_stem` (блюёт), что даёт корректные формы.
- **домандякаться** (`-ка-`), именная деривация, падежная парадигма — вне движка.
- **обо-** — третий алломорф приставки о-/об-, обязателен перед стечением согласных
  (обосраться, обоссать, обдристать, ободрать); движок его не имеет, а о-/об- применяет по
  правилу предлога (упрощение). Формы с обо- источник свидетельствует, движок строит
  вместо них о-формы (осрать, оссать, одристать). Обоснование:
  `docs/decisions/o-ob-allomorphy.md`.

## Связь classic-режима с таксономией

`Mode::Classic` (обратно совместимые 9 корней) выводится ИЗ таксономии, без хардкода
имён: `Nuclear` ∪ {`Excretory` с продуктивностью ≤ B}. Класс B среди экскреторных —
только *ср-*, *сс-*. Итого ядро(7) + {сра, сса} = 9.
